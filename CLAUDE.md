# Codebase map — TikTok-Reels

Local web editor that assembles 4–6 mp4 clips into one vertical reel with trim, editable karaoke captions, and a preset-styled heading+subheading block, exported to 1080×1920 mp4. See `docs/superpowers/specs/2026-07-09-first-reel-design.md` and `docs/superpowers/plans/2026-07-09-first-reel.md`.

## Run commands

- Tests: `.venv/Scripts/python -m pytest -q`
- Server: `.venv/Scripts/python -m uvicorn app.main:app --reload` (then open http://127.0.0.1:8000)
- Setup: `python -m venv .venv && .venv/Scripts/pip install -e .[dev]`
- Requires `ffmpeg`/`ffprobe` on PATH for clip probing/export (not required for `pytest`, which mocks subprocess calls).

## Conventions

- No JS build step/bundler — icon SVGs are hand-inlined directly in markup. When an icon is needed, use [Lucide](https://lucide.dev) icon paths (copy the `<path>` markup from lucide.dev; keep the existing `viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"` wrapper style already used for play/pause/restart/step/bold/italic/underline icons).

## Inventory

- `app/models.py` — Pydantic entities: `Project`, `ClipLayer`, `TextPreset` (`font` defaults to `"Public Sans"`, constrained by the UI to the 2 vendored families; `bold`/`italic`/`underline: bool = False`, whole-block formatting, added 2026-07-14), `TextBlockLayer` (single `heading` line, may contain `\n` for multiline — `subheading` was dropped 2026-07-10), `CaptionWord`, `CaptionTrack`, `new_id()`.
- `app/store.py` — JSON persistence: `save_project`, `load_project`, `save_preset`, `load_presets`.
- `app/media.py` — `ffprobe_cmd`, `probe_duration`, `media_response` (serves a local file via FastAPI, 404s if missing), `run_export` (runs an ffmpeg command, raises `RuntimeError` with stderr on failure), `pick_file` (opens a native OS file-open dialog, returns the chosen path or `None`). Both `probe_duration` and `run_export` resolve `ffprobe`/`ffmpeg` from a freshly-read registry PATH rather than the process's inherited env, so a PATH change (e.g. installing ffmpeg) takes effect without restarting every ancestor process.
- `app/main.py` — FastAPI composition root: `GET /`, `POST/GET/PUT /api/projects[/{id}]`, `GET /api/probe`, `GET /api/pick-file`, `GET /media`, `POST /api/projects/{id}/export`, static mount at `/static`.
- `app/timeline.py` — `ordered`, `clip_duration`, `sequence_duration`, `locate` (timeline time -> clip + source-time); mirrored in `static/preview.js`.
- `app/ffmpeg_cmd.py` — `build_export_cmd` (per-clip trim/scale/pad, concat, optional ASS burn-in), `escape_filter_path`.
- `app/ass_render.py` — `render_ass(project, presets) -> str` (full ASS file: `[Script Info]`/`[V4+ Styles]`/`[Events]` for each text block), `ass_time(seconds) -> str`, `hex_to_ass(hex) -> str` (AABBGGRR). Style line's Bold/Italic/Underline columns come from `TextPreset.bold/italic/underline` (added 2026-07-14). Text-block dialogue: `\pos` anchor, `\fad`+`\t` scale pop for `entrance="fade_pop"`, literal `\n` in `heading` becomes ASS's `\N` hard line break (2026-07-14). (`subheading`/`\N` merge dropped 2026-07-10 — one heading line per block.)
- `static/timeline.js` — `render(project, timelineTime, selected, onSelect)` (ruler, playhead, clip/text/caption blocks, toolbar time readout, SLICE button position, AUDIO placeholder), `tick(timelineTime)` (cheap playhead/SLICE-button/time-readout-only update, driven every animation frame during playback by editor.js so motion stays smooth between the heavier `render()` calls), `groupWords(words, max)` (group caption words), `timeAtX(clips, rulerRect, clientX)` (timeline coordinate math). Depends on Preview (preview.js). Clicking a block calls `editor.js`'s `onTimelineSelect`, which opens the matching `#style-panel` context section (VIDEO/TEXT/CAPTIONS). `#timeline-strip` now has a toolbar (zoom −/+ buttons, non-functional; mm:ss.s time readout) above the ruler, a `.slice-btn` (visual only — no slice feature yet, added 2026-07-14) that repositions to stay above the playhead as time changes, and a fourth AUDIO row rendering a static dummy waveform (deterministic pseudo-random bars, regenerated only when track width changes) since there's no real audio-track data yet. TEXT and CAPTIONS rows share the same 44px height so their blocks render at a consistent size.
- `static/seed.js` — `seedDefaults(project)`: seeds one sample caption line (only) when `project.captions` is null, so the CAPTIONS timeline row isn't empty on a fresh project. Does not seed a text block — `editor.js`'s `ensureTextBlock()`/`textPreset` already provide a real one.
- `static/css/tokens.css` — design tokens (colors, fonts, spacing, radius) per `docs/superpowers/specs/2026-07-10-design-foundation-design.md`; every later screen builds on this.
- `static/ui-components.js` — `window.UI.buttonGroup(container, options, activeValue, onSelect)`: renders a row of toggle buttons with `aria-pressed` state, exactly one active; returns a `setActive(value)` updater. Used for TEXT ALIGN and the two POSITION rows; reusable for any future single-select control (e.g. presets, when Task 8 adds them). `window.UI.numberField(container, {label, unit, value, step, min, max, onChange})`: renders a `.style-field`-styled labeled number input plus a custom up/down stepper (`.number-field-wrap`/`.number-field-stepper`; the native OS spin control turned out to be unstylable — confirmed via computed-style inspection — so it's hidden globally in base.css and replaced by this CSS-drawn one), wires typing and stepper clicks to `onChange(number)`, returns a `setValue(v)` updater. Used for START/END/WIDTH/OFFSET H/OFFSET V — one owner for that markup instead of copy-pasted `<label class="style-field">` blocks (a duplication that caused a real styling bug once, fixed by extracting this). `window.UI.colorSwatch(container, {label, value, onChange})`: renders a small square `.color-swatch` color input with a mixed-case `.color-swatch-label` beside it (one field per row — never shares a row with another control), wires picks to `onChange(hexString)`, returns a `setValue(hex)` updater. Used for Color/Outline/Box Color. `window.UI.accordion(header, body, {expanded})`: wires an existing header `<button>` + body `<div>` pair, toggling `body.hidden` and the header's `aria-expanded` attribute on click. Used for the TEXT panel's MISC section (`#text-misc-header`/`#text-misc-body`, added 2026-07-15).
- `static/css/components/button-group.css` — `.btn-group` (the toggle-row layout) + `.icon-btn` (small square icon buttons, flex-centered; used by the disabled caption-toolbar placeholder and the TEXT panel's Bold/Italic/Underline toggle buttons, with an `[aria-pressed="true"]` highlight rule shared with `.btn-group button`).
- `static/css/components/number-field.css` — `.number-field-wrap`/`.number-field-stepper`/`.number-field-step` (CSS-triangle up/down buttons); pairs with `UI.numberField`.
- `static/css/components/color-swatch.css` — `.color-swatch`: apply to any `<input type="color">` to make it fill its box with solid color, no inset border/padding (native color inputs otherwise render with browser chrome around a small swatch). Paired with `UI.colorSwatch()` below — that's the call site to reach for anywhere a color is picked (background, text, border, etc.).
- `static/css/components/style-panel.css` — `#style-panel` (right aside, 320px, `hidden` by default so Center reclaims the width): `#style-panel-close` (×, top-right, always visible when open) plus three `.context-panel` sections shown one at a time — `#panel-video` (TRIM in/out + ORDER move up/down, new), `#panel-captions` (a `.caption-placeholder` block: disabled B/I/U/align icon buttons + a static `.caption-preview-box`, clearly labeled "COMING IN A LATER TASK" — non-functional visual stand-in for Task 10/11's real caption editor), `#panel-text` (`.style-group`/`.style-group-label`/`.style-row`/`.style-field`/`.style-divider` sections: a multiline `#text-heading` `<textarea>`, TIME, STYLE — Word-toolbar-style FONT `<select>` + SIZE `UI.numberField` + Bold/Italic/Underline `.icon-btn` toggles (SVG icons) — TEXT ALIGN, POSITION; added 2026-07-14). Matches the north-star mockup (`docs/superpowers/specs/assets/2026-07-10-design-foundation-mockup.html`). Preset swatches (CLEAN/BOXED/POP/MINIMAL) from the mockup are deliberately not built yet — deferred to Task 8.
- `static/editor.js` — text-block wiring: `defaultTextPreset()`/`loadTextPreset(id)`/`saveTextPreset()` (client-only TextPreset, persisted in `localStorage` under `textPreset:<projectId>` until Task 8 adds a presets API), `ensureTextBlock()` (lazily creates the single `project.text_blocks[0]`), `updateTextBlock()`/`updateTextStyle()` (input handlers -> save + re-render), `renderTextPanel()` (populate controls + button groups from state on load, incl. FONT select, SIZE `UI.numberField`, and `wireTextStyleToggle(id, prop)` for the Bold/Italic/Underline buttons' `aria-pressed` state). Position is a `posRow`/`posCol` anchor grid (thirds of the 1080x1920 canvas, `POSITION_ANCHORS_X/Y`) plus an `offsetX`/`offsetY` pixel nudge; `computeXY()` derives `TextPreset.x/y` from those — the anchor/offset split is UI-only, not part of the persisted model. `showPanel(type)`/`closePanel()` toggle `#style-panel`'s `hidden` attribute and switch which `.context-panel` section is visible; `selected` (`{type, item}`) tracks the current clip/text/caption selection, driving both the context panel and the highlight state in `renderClipList()`/`Timeline.render()`. `selectClip(c)`/`renderVideoPanel(c)` populate `#panel-video`'s trim (`UI.numberField`, reusing `clampTrim`) and reorder controls for the selected clip. `renderClipList()` renders display-only thumbnail+name+duration rows (`formatClipDuration`, `mm:ss.s`), click-to-select — trim/reorder UI lives in the context panel, not inline. `setPanelCollapsed(bool)` toggles `#panel.collapsed` (72px icon rail) and persists to `localStorage` (`panelCollapsed`). `setSafeZonesVisible(bool)` toggles `#safe-zones`'s `hidden` attribute + `#safe-zones-toggle`'s `aria-pressed`, persists to `localStorage` (`safeZonesVisible`); same shape as `setPanelCollapsed`, added 2026-07-15.
- `static/preview.js` — `Preview.renderText(project, presets, timelineTime)` composites one `.text-block` div (may be multiline via `\n`, rendered with the stage CSS's existing `white-space: pre`) per visible block into `#overlay` (position/size scaled from the 1080x1920 canvas to the stage's actual pixel size; outline via `-webkit-text-stroke` or `box`/`box_color` background; `font-family`/`font-weight`/`font-style`/`text-decoration` set from `preset.font`/`bold`/`italic`/`underline`, added 2026-07-14); `Preview.currentTimelineTime()` computes the current timeline time fresh from `player.currentTime`/`activeIndex` on every call (not a cache) so editor.js's animation-frame loop always reads a live value. `Preview.play()`/`Preview.pause()`/`Preview.restart()` (added 2026-07-14) back the transport's `#play-pause` toggle button, `#restart` button, and editor.js's Up-arrow (toggle)/Down-arrow (restart) shortcuts; `play()` keeps the old "restart from clip 0 if at the end" behavior, `restart()` always jumps to clip 0 and plays. `#play-pause` contains two SVGs (`.icon-play`/`.icon-pause`) whose `style.display` is toggled directly (not the `hidden` attribute, which browsers don't reliably map to `display: none` on SVG elements — both icons stayed visible at once) by the video element's native `play`/`pause` events, so the icon stays correct no matter what triggered the state change. `#time`'s numeric readout is hidden via CSS (`stage.css`) since the timeline toolbar shows the time now, but the element still holds the value as `textContent` — editor.js's `nudgeTime`/`renderTimeline` still read it.
