# Codebase map ‚Äî TikTok-Reels

Local web editor that assembles 4‚Äì6 mp4 clips into one vertical reel with trim, editable karaoke captions, and a preset-styled heading+subheading block, exported to 1080√ó1920 mp4. See `docs/superpowers/specs/2026-07-09-first-reel-design.md` and `docs/superpowers/plans/2026-07-09-first-reel.md`.

## Run commands

- Tests: `.venv/Scripts/python -m pytest -q`
- Server: `.venv/Scripts/python -m uvicorn app.main:app --reload` (then open http://127.0.0.1:8000)
- Setup: `python -m venv .venv && .venv/Scripts/pip install -e .[dev]`
- Requires `ffmpeg`/`ffprobe` on PATH for clip probing/export (not required for `pytest`, which mocks subprocess calls).

## Conventions

- No JS build step/bundler ‚Äî icon SVGs are hand-inlined directly in markup. When an icon is needed, use [Lucide](https://lucide.dev) icon paths (copy the `<path>` markup from lucide.dev; keep the existing `viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"` wrapper style already used for play/pause/restart/step/bold/italic/underline icons).
- Reusable JS logic ‚Äî UI components (`window.UI.*`) and API/service calls (`window.Api.*`) ‚Äî each live in their own file, one function/component per file; never grouped into a shared `*-components.js`-style catch-all.
- Every `static/*.js` and `static/css/**/*.css` file opens with a one- or two-line comment stating that file's purpose/role, so its job is clear without reading the whole file. Keep this comment current when a file's role changes.

## Inventory

- `app/models.py` ‚Äî Pydantic entities: `MediaItem(id, file_path, duration)` ‚Äî separates imported media library from timeline clip references; `Project` (`media_library: list[MediaItem] = []`, added 2026-07-15), `ClipLayer` (`media_id: str` required field linking to MediaItem, added 2026-07-15), `TextPreset` (`font` defaults to `"Public Sans"`, constrained by the UI to the 2 vendored families; `bold`/`italic`/`underline: bool = False`, whole-block formatting, added 2026-07-14); `box_width_mode`/`box_height_mode: str = "fit"` + `box_width`/`box_height: int` (independent per-axis fixed-vs-fit-to-content sizing), `box_background: bool`/`box_background_color: str` (replaces the old `box`/`box_color`, migrated on load), `box_border_width`/`box_border_color`/`box_border_radius` (the Text Box component, added 2026-07-17), `usage_count: int = 0` (drives the STYLE accordion's most-used list, added 2026-07-18), `TextBlockLayer` (single `heading` line, may contain `\n` for multiline ‚Äî `subheading` was dropped 2026-07-10), `CaptionWord`, `CaptionTrack`, `new_id()`.
- `app/store.py` ‚Äî JSON persistence: `save_project`, `load_project`, `save_preset`, `load_presets`.
- `app/media.py` ‚Äî `ffprobe_cmd`, `probe_duration`, `media_response` (serves a local file via FastAPI, 404s if missing), `run_export` (runs an ffmpeg command, raises `RuntimeError` with stderr on failure), `pick_file` (opens a native OS file-open dialog, returns the chosen path or `None`). Both `probe_duration` and `run_export` resolve `ffprobe`/`ffmpeg` from a freshly-read registry PATH rather than the process's inherited env, so a PATH change (e.g. installing ffmpeg) takes effect without restarting every ancestor process.
- `app/main.py` ‚Äî FastAPI composition root: `GET /`, `POST/GET/PUT /api/projects[/{id}]`, `GET /api/probe`, `GET /api/pick-file`, `GET /media`, `POST /api/projects/{id}/export`, GET/POST /api/presets, static mount at `/static`.
- `app/timeline.py` ‚Äî `ordered`, `clip_duration`, `sequence_duration`, `locate` (timeline time -> clip + source-time); mirrored in `static/preview.js`.
- `app/ffmpeg_cmd.py` ‚Äî `build_export_cmd` (per-clip trim/scale/pad, concat, optional ASS burn-in), `escape_filter_path`.
- `app/ass_render.py` ‚Äî `render_ass(project, presets) -> str` (full ASS file: `[Script Info]`/`[V4+ Styles]`/`[Events]` for each text block), `ass_time(seconds) -> str`, `hex_to_ass(hex) -> str` (AABBGGRR). Style line's Bold/Italic/Underline columns come from `TextPreset.bold/italic/underline` (added 2026-07-14). Text-block dialogue: `\pos` anchor, `\fad`+`\t` scale pop for `entrance="fade_pop"`, literal `\n` in `heading` becomes ASS's `\N` hard line break (2026-07-14). (`subheading`/`\N` merge dropped 2026-07-10 ‚Äî one heading line per block.) As of 2026-07-17, `_style()` no longer switches `BorderStyle` based on a box toggle (always plain outline); box background/border render as a separate vector-drawn (`\p1`) `Dialogue` line emitted by `_box_dialogue()` immediately before each block's text dialogue, sized/word-wrapped via `_wrapped_lines_and_size()` (uses `app/font_metrics.py`), with rounded corners from `_rounded_rect_path()`.
- `app/font_metrics.py` ‚Äî `wrap_text(text, measure_width, max_width_px) -> str` (pure word-wrap, injectable measurer), `pil_font_measurer(font_name, size_px) -> Callable[[str], float]` (decompresses the vendored `.woff2` via fontTools to sfnt bytes in-memory, measures with Pillow ‚Äî used only by `ass_render.py`'s export path, added 2026-07-17).
- `static/timeline.js` ‚Äî `render(project, timelineTime, selected, onSelect)` (ruler, playhead, clip/text/caption blocks, toolbar time readout, SLICE button position, AUDIO placeholder), `tick(timelineTime)` (cheap playhead/SLICE-button/time-readout-only update, driven every animation frame during playback by editor.js so motion stays smooth between the heavier `render()` calls), `groupWords(words, max)` (group caption words), `timeAtX(clips, rulerRect, clientX)` (timeline coordinate math). Depends on Preview (preview.js). Clicking a block calls `editor.js`'s `onTimelineSelect`, which opens the matching `#style-panel` context section (VIDEO/TEXT/CAPTIONS). `#timeline-strip` now has a toolbar (zoom ‚àí/+ buttons, non-functional; mm:ss.s time readout) above the ruler, a `.slice-btn` (visual only ‚Äî no slice feature yet, added 2026-07-14) that repositions to stay above the playhead as time changes, and a fourth AUDIO row rendering a static dummy waveform (deterministic pseudo-random bars, regenerated only when track width changes) since there's no real audio-track data yet. TEXT and CAPTIONS rows share the same 44px height so their blocks render at a consistent size.

- `static/api-list-presets.js` ‚Äî `Api.listPresets() -> Promise<TextPreset[]>`: `GET /api/presets`, fetches the global saved-style preset library.
- `static/api-save-preset.js` ‚Äî `Api.savePreset(preset) -> Promise<TextPreset>`: `POST /api/presets`, saves a new or updates an existing saved-style preset.
- `static/text-panel-style.js` ‚Äî `TextPanel.renderStyle()`/`TextPanel.loadSavedPresets()`: the TEXT panel's STYLE accordion ‚Äî a saved-style preset library distinct from `project.text_presets` (per-block live working style), persisted via `Api.listPresets()`/`Api.savePreset()`. `loadSavedPresets()` fetches the library once at startup (before the first `renderTextPanel()`); `renderStyle()` renders a "+ Save current style" button (prompts for a name, snapshots the current preset's style fields via `styleFieldsOf()` ‚Äî everything except identity/stage-placement/usage-stats ‚Äî into a new saved preset with `usage_count: 0`) plus a top-3 "most used" inline list (`#text-style-most-used`, sorted by `usage_count`) and a `UI.settingsRow` ("Browse all styles") that opens a full drill-down (`#panel-text-style`, `UI.subPanelHeader`) listing every saved style sorted by `usage_count`. Clicking any saved-style row (`renderStyleListRow()`) calls `applySavedPreset()`: copies the saved style's fields onto the current block's live preset, increments and persists the saved preset's `usage_count`, saves the project, re-renders the TEXT panel, and closes the drill-down.
