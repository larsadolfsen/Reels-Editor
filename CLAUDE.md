# Codebase map — TikTok-Reels

Local web editor that assembles 4–6 mp4 clips into one vertical reel with trim, editable karaoke captions, a preset-styled heading block, and picture-in-picture video boxes, exported to 1080×1920 mp4. Multi-project: a full-screen picker at cold start plus an in-editor PROJECTS panel (open/rename/duplicate/delete). See `docs/superpowers/specs/2026-07-09-first-reel-design.md` and `docs/superpowers/plans/2026-07-09-first-reel.md`.

## Run commands

- Tests: `.venv/Scripts/python -m pytest -q`
- Server: `.venv/Scripts/python -m uvicorn app.main:app --reload` (then open http://127.0.0.1:8000)
- Setup: `python -m venv .venv && .venv/Scripts/pip install -e .[dev]`
- Requires `ffmpeg`/`ffprobe` on PATH for clip probing/export (not required for `pytest`, which mocks subprocess calls).

## Conventions

- No JS build step/bundler — icon SVGs are hand-inlined directly in markup. When an icon is needed, use [Lucide](https://lucide.dev) icon paths (copy the `<path>` markup from lucide.dev; keep the existing `viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"` wrapper style already used for play/pause/restart/step/bold/italic/underline icons).
- Reusable JS logic — UI components (`window.UI.*`) and API/service calls (`window.Api.*`) — each live in their own file, one function/component per file; never grouped into a shared `*-components.js`-style catch-all.
- Prefer small units that each own one feature or function over broad shared abstractions — microcomponents on the frontend, small focused modules on the backend (e.g. `media.py`/`timeline.py`/`ass_render.py`/`font_metrics.py`/`ffmpeg_cmd.py` staying separate rather than merging). Default to a new small file rather than extending an existing multi-purpose one.
- Every `static/*.js` and `static/css/**/*.css` file opens with a one- or two-line comment stating that file's purpose/role, so its job is clear without reading the whole file. Keep this comment current when a file's role changes.
- No inline `style="..."` attributes in `static/index.html` or JS-rendered markup — all styling lives in `static/css/**` component files as classes, even for small one-off tweaks.

defaults `True` so existing saved projects behave unchanged until re-probed) is consumed by `app.ffmpeg_cmd.build_export_cmd` to decide whether a clip needs synthesized silent audio; `Project` (`media_library: list[MediaItem] = []`, added 2026-07-15; `name`/`created_at`/`updated_at` + `video_boxes: list[VideoBoxLayer] = []`, multi-project work), `ProjectSummary(id, name, created_at, updated_at)` (lightweight `GET /api/projects` list payload — never the full project), `ClipLayer` (`media_id: str` required field linking to MediaItem, added 2026-07-15), `VideoBoxLayer` (picture-in-picture layer: `media_id`/`file_path`, `in_point`/`out_point` source trim, `start` timeline seconds — end is always derived as `start + out_point - in_point` — `x`/`y`/`width`/`height` px on the 1080×1920 canvas with height set from source aspect at creation and kept locked on resize, `z_index: int = -1` so new boxes default just below the default text z_index 0), `TextPreset` (`font` defaults to `"Public Sans"`, constrained by the UI to the 2 vendored families; `weight: int = 400` — 400/500/600/700, replaces the old `bold: bool` 2026-07-19, migrated on load — `italic`/`underline: bool = False`, whole-block formatting, added 2026-07-14); `box_width_mode`/`box_height_mode: str = "fit"` + `box_width`/`box_height: int` (independent per-axis fixed-vs-fit-to-content sizing), `box_background: bool`/`box_background_color: str` (replaces the old `box`/`box_color`, migrated on load), `box_border_width`/`box_border_color`/`box_border_radius` (the Text Box component, added 2026-07-17), `usage_count: int = 0` (drives the STYLE accordion's most-used list, added 2026-07-18), `highlight_color`/`highlight_mode`/`max_words_per_line` (caption karaoke highlight color/mode and line-grouping size; `highlight_color` is now shared with rich-text formatting too, see below), `highlight: bool = False` (block-level rich-text highlight default, off — added 2026-07-19); `FormatRun` (Phase 5 rich-text formatting, added 2026-07-19: `start`/`end` character offsets into a `TextBlockLayer.heading` string, plus sparse-optional style overrides — `font`/`size_px`/`color`/`outline_color`/`outline_px`/`weight`/`italic`/`underline`/`highlight`/`highlight_color`, each `None` by default so an unset field falls through to the block's base `TextPreset`); `TextBlockLayer` (single `heading` line, may contain `\n` for multiline — `subheading` was dropped 2026-07-10 — `formatting_runs: list[FormatRun] = []` added 2026-07-19, additive/backward-compatible: an empty list renders and exports identically to before Phase 5; `z_index: int = 0` for cross-layer stacking, see the LAYERS panel), `CaptionWord`, `CaptionTrack` (`z_index: int = 0`; `preset_id: str` links a caption track to a `TextPreset`, same pattern as `TextBlockLayer.preset_id`, defaulted via `new_id()` so old saved projects self-heal instead of failing validation, added 2026-07-19), `new_id()`.
- `app/store.py` — JSON persistence: `save_project`, `load_project`, `list_projects` (loads every `<data>/projects/*.json`), `delete_project`, `save_preset`, `load_presets`.
- `app/media.py` — `ffprobe_cmd`, `probe_duration`, `has_audio_stream(path) -> bool` (Phase 6, added 2026-07-20: ffprobes for an audio stream via `-select_streams a`, used to populate `MediaItem.has_audio` at import time and exposed through `GET /api/probe`), `media_response` (serves a local file via FastAPI, 404s if missing), `run_export` (runs an ffmpeg command, raises `RuntimeError` with stderr on failure), `pick_file` (opens a native OS file-open dialog, returns the chosen path or `None`). `probe_duration`, `has_audio_stream`, and `run_export` resolve `ffprobe`/`ffmpeg` from a freshly-read registry PATH rather than the process's inherited env, so a PATH change (e.g. installing ffmpeg) takes effect without restarting every ancestor process.
- `app/main.py` — FastAPI composition root: `GET /`, `POST/GET/PUT /api/projects[/{id}]`, `GET /api/projects` (all projects as `ProjectSummary[]`), `DELETE /api/projects/{pid}` (204), `POST /api/projects/{pid}/duplicate` (new id, name `"<name> copy"`), `GET /api/probe` (returns `{duration, has_audio}` as of Phase 6, 2026-07-20 — `has_audio` populated via `media.has_audio_stream`), `GET /api/pick-file`, `GET /media`, `POST /api/projects/{id}/export`, GET/POST /api/presets, `GET /api/fonts/{name}/weights` (added 2026-07-19), `POST /api/projects/{pid}/transcribe` (added 2026-07-19), static mount at `/static`. `export_project` now also burns in karaoke captions (added 2026-07-19): when `p.captions.words` is non-empty it renders `ass_render.render_caption_ass(p, caption_preset)` to a `-captions.ass` file up front and passes it as `build_export_cmd(..., caption_ass_path=...)` on both the banded (`video_boxes`) and plain (`ass_path`) export paths, so captions always burn in as the topmost/last layer over any text-block or video-box output.
- `app/timeline.py` — `ordered`, `clip_duration`, `sequence_duration`, `locate` (timeline time -> clip + source-time); mirrored in `static/preview.js`.
- `app/ffmpeg_cmd.py` — `build_export_cmd` (per-clip trim/scale/pad, concat, optional ASS burn-in with a `fontsdir=static/fonts` option so libass resolves the generated per-weight font files by exact name instead of depending on system-installed fonts, added 2026-07-19), `escape_filter_path`, also `build_audio_cmd(project, wav_path)` — audio-only concat export (mono, 16kHz) used to produce a timeline-relative wav for transcription (**known gap, not yet fixed:** unlike `build_export_cmd`, this still assumes every clip has an audio stream — a video-only clip will break transcription's audio export; tracked as Phase 6 follow-up). As of Phase 6 (2026-07-20), `build_export_cmd` synthesizes silent audio for clips whose `MediaItem.has_audio` is `False`: an `-f lavfi -i anullsrc=channel_layout=stereo:sample_rate=44100` input is inserted at that clip's position instead of reading `[i:a]` from the (nonexistent) real audio stream. Input indices are tracked via a running `input_index` counter (not just enumerate index `i`) so interleaved synthesized-silence inputs don't collide with subsequent clips' or video boxes' real input indices — `next_input_index` in the `bands` branch is seeded from this counter rather than `len(clips)`. `build_export_cmd` also accepts `caption_ass_path` (added 2026-07-19): when given, it chains one more `ass` filter as the always-final stage, after any `ass_path`/`bands` output (`[vo]`/`[vc]`/the last band's label), producing `[vcap]`, which becomes the final `-map` target — so karaoke captions always render on top of any text-block or video-box layers.
- `app/transcribe.py` — `words_from_segments(segments) -> list[CaptionWord]` (pure), `transcribe_file(path) -> list[CaptionWord]` (lazy `WhisperModel("large-v3", device="cuda")` load, module-level cache). Requires the `ml` optional dependency group (`.venv/Scripts/pip install -e .[ml]`).
- `app/ass_render.py` — `render_ass(project, presets) -> str` (full ASS file: `[Script Info]`/`[V4+ Styles]`/`[Events]` for each text block), `ass_time(seconds) -> str`, `hex_to_ass(hex) -> str` (AABBGGRR). Style line's Italic/Underline columns come from `TextPreset.italic/underline`; the Bold column is unconditionally `0` and `Fontname` instead includes a weight label (e.g. `"Public Sans Medium"`) matching one of the static per-weight files `scripts/generate_font_weights.py` generates, since ASS has no numeric font-weight concept and libass selects fonts purely by family name (added 2026-07-14, weight-aware 2026-07-19). Text-block dialogue: `\pos` anchor, `\fad`+`\t` scale pop for `entrance="fade_pop"`, literal `\n` in `heading` becomes ASS's `\N` hard line break (2026-07-14). Phase 5 rich-text formatting (added 2026-07-19): `renderText()` no longer sets `div.textContent`/`div.style.color`/`fontFamily`/`fontWeight`/`fontStyle`/`textDecoration`/`webkitTextStroke` at the div level — instead it splits `block.heading` into one `<span class="text-run">` per `formatting_runs` boundary segment (a block with `formatting_runs: []` produces exactly one span covering the whole string, styled identically to the old div-level rendering), each span independently resolving `color`/`fontFamily`/`fontSize`/`fontWeight`/`fontStyle`/`textDecoration`/`webkitTextStroke`(width+color)/highlight-`backgroundColor` from `(run && run.X) || preset.X` (or `!= null` for boolean fields, so an explicit `false` override isn't lost) — multi-line highlight backgrounds come for free from the browser's native inline-span line-wrapping, no extra CSS needed. `Preview.getActiveFormatSelection() -> {blockId, start, end} | null` tracks the current non-collapsed text selection on the selected block (via `UI.textInteraction`'s new `onSelectionChange` callback, see `ui-text-selection.js`/`ui-text-interaction.js` below), cleared when the selected block changes or the selection collapses back to a caret (the latter detected via `onEditStart`, which only fires on a plain non-drag click). `static/text-panel-font-style.js`/`static/text-panel-font-weight.js` read this to decide whether a FONT-accordion edit should write a `FormatRun` for the active range instead of the base preset.
